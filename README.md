git-репозиторий
 Создайте файл README.md с
описанием проекта
 Создайте следующие ветки
 master (2 коммита)
 dev из ветки master (2 коммита из
master + 2 дополнительных)
 features/dosmth (коммиты из dev + 1
дополнительный)
 hotfix/bugfix (2 коммита из master + 1
дополнительный)
Разыграйте следующие сценарии:
 Релиз

попасть в master
 Выпуск исправления
Изменения из hotfix/bugfix должны
сначала попасть в master, а потом и в
остальные ветки
